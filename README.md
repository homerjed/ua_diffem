# UA-DiffEM

This folder contains a deliberately small example of using an uncertainty-aware
flow matching posterior inside the DiffEM loop from
[`arXiv:2510.12691`](https://arxiv.org/abs/2510.12691).

DiffEM is used here as:

1. E-step: sample clean reconstructions `x_hat ~ p_theta(x | y_obs)` from the
   current conditional UA-flow posterior.
2. M-step: train the same posterior on pairs `(x_hat, y_tilde)`, where
   one fixed `y_tilde ~ Q(. | x_hat)` dataset is generated by the known
   corruption channel for that M-step.

The initial E-step can bootstrap from the corrupted observation because an
untrained posterior sampler is not useful yet. Subsequent E-steps use UA-flow
posterior samples and preserve the propagated uncertainty maps.

The observed corrupted MNIST dataset `y_obs` is sampled once before training and
then reused for every E-step; it is not regenerated at each iteration or epoch.

## Files

- `uncertainty_aware_flow.py`: UA-flow posterior builders. The default model is
  a spatially conditioned UNet that predicts velocity mean and log-sigma.
- `diffem.py`: DiffEM-specific pieces: the known corruption channel, E-step,
  M-step, optimizer, EMA tracking, and checkpoint helpers.
- `models/dit.py`: a compact optional image-conditioned DiT-style posterior network.
- `train_mnist.py`: a runnable MNIST inpainting/noisy-observation example.
- `train_shear.py`: a runnable toy weak-lensing shear inversion example.
- `train_spherical.py`: spherical weak-lensing DiffEM on HEALPix maps using
  GLASS, healpy, and a DeepSphere-style velocity network.
- `train_spherical_gaussian.py`: same spherical setup with a Gaussian
  convergence prior, so the latent prior and shear/noise observation model are
  linear-Gaussian.
- `shear.py`: shared synthetic-shear dataset and Kaiser-Squires utilities.
- `shear/glass_maps.py`: GLASS/HEALPix spherical map generation and spherical
  shear corruption utilities.
- `models/spherical_unet.py`: DeepSphere-style Chebyshev graph-convolution U-Net for
  UA-flow velocity prediction on NESTED HEALPix maps.
- `notes/`: introductory planar weak-lensing notes plus a separate spherical
  weak-lensing simulation note, with reproducible shear figures.
- `shear_tests/`: evaluation folders for saved shear runs.

## Run

```bash
python -m ua_diffem.train_mnist
```

Useful quick smoke run:

```bash
python -m ua_diffem.train_mnist \
  --train_size 32 \
  --em_steps 1 \
  --m_steps_per_em 1 \
  --batch_size 4 \
  --posterior_batch_size 4 \
  --posterior_sample_steps 1
```

Try the compact DiT posterior:

```bash
python -m ua_diffem.train_mnist --model_name dit --model_dim 128 --attn_heads 4
```

Run a small spherical weak-lensing smoke job:

```bash
python -m ua_diffem.scripts.train_spherical \
  --train_size 8 \
  --nside 8 \
  --em_steps 1 \
  --m_steps_per_em 1 \
  --batch_size 2 \
  --posterior_batch_size 2 \
  --posterior_sample_steps 1
```

Run the linear-Gaussian spherical variant:

```bash
python -m ua_diffem.scripts.train_spherical_gaussian \
  --train_size 8 \
  --nside 8 \
  --em_steps 1 \
  --m_steps_per_em 1 \
  --batch_size 2 \
  --posterior_batch_size 2 \
  --posterior_sample_steps 1
```

The same trainer also accepts `--prior_family gaussian` or
`--prior_family lognormal`; the Gaussian entry point just sets the default run
folder, prior family, CAMB-backed convergence power spectrum, and a smaller
physical shear-noise level for convenience. The default spherical script uses a
toy spectrum
`C_ell = A (ell + 1)^(-spectral_index) exp[-damping ell (ell + 1)]`; pass
`--kappa_power_spectrum camb` to either script for a cosmological weak-lensing
convergence spectrum from CAMB and a Smail source distribution. The exact
`C_ell` used for a run is saved to `out/kappa_power_spectrum.npz`.

For the spherical example, build the CUDA/GLASS/healpy runtime with:

```bash
./scripts/build-spherical-sif.sh
./scripts/run-spherical-singularity.sh python -m ua_diffem.scripts.train_spherical --train_size 8 --em_steps 1
```

`--covariance_mode zero` skips the covariance term in UA-flow uncertainty
propagation during sampling. `--covariance_mode jvp` estimates that term with
JVP probes; it is slower, but gives a fuller propagated uncertainty estimate.

When multiple local JAX devices are visible, `train_mnist.py` enables
data-parallel sharding by default: model and optimizer arrays are replicated
across devices, and M-step batches are sharded along the leading batch
dimension. Disable it with `--no-data_parallel_sharding`.

Each M-step now builds a fixed train/validation split from the reconstructed
dataset and evaluates validation loss every `--m_step_validation_freq` updates.
M-step early stopping is available but disabled by default. Enable it with
`--m_step_early_stopping`; it monitors validation loss after
`--m_step_min_steps` and stops after `--m_step_patience` validation checks
without an improvement larger than `--m_step_min_delta`.

Outputs go to `runs/ua_diffem/mnist_basic/` by default:

- `config.json`
- `out/losses.npy`, `out/losses_by_em.npz`,
  `out/validation_losses_by_em.npz`, and `out/losses_by_em.png`
- `states/state.eqx` and `states/state_em_XX.eqx`
- `reconstructions/reconstructions_em_XX.png` preview panels with truth,
  measurement, reconstruction, and uncertainty rows

## Shear Evaluation Folders

Saved shear runs can now be evaluated by run name. Each script creates a
same-named folder inside `runs/ua_diffem/<run_name>/` and writes plots and
metrics there.

```bash
python -m ua_diffem.shear_tests.run_all --run_name shear_basic
python -m ua_diffem.shear_tests.quantitative_reconstruction.run --run_name shear_basic
python -m ua_diffem.shear_tests.visual_examples.run --run_name shear_basic
python -m ua_diffem.shear_tests.uncertainty_aware.run --run_name shear_basic
python -m ua_diffem.shear_tests.weak_lensing_statistics.run --run_name shear_basic
```

The current folders are:

- `quantitative_reconstruction/`: reconstruction and forward-model metrics,
  plus a Kaiser-Squires baseline comparison
- `visual_examples/`: qualitative truth / observed shear / baseline /
  reconstruction / uncertainty panels
- `uncertainty_aware/`: interval coverage, reliability, and risk-coverage
  diagnostics
- `weak_lensing_statistics/`: convergence power-spectrum and peak-count
  recovery plots
